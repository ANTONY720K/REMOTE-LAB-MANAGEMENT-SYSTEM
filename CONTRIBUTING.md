# Contributing to Remote Lab Management System

Thank you for your interest in contributing to the Remote Lab Management System! We appreciate your help in improving this project. Please follow the guidelines below to ensure a smooth contribution process.

## Contributing Guidelines
- Read the [README.md](README.md) for an overview of the project and how it works.
- Ensure your contributions align with the project's goals and objectives.

## Code Style Requirements
- Follow the coding conventions used in the existing codebase. Specifically:
  - Use meaningful variable names.
  - Keep functions short and focused on a single task.
  - Maintain proper indentation and spacing for readability.

## Commit Message Standards
- Use the following format for commit messages:
  - **Type:** A short descriptor of the change (e.g., `fix`, `feature`, `docs`, `style`).
  - **Scope:** Optional, the part of the project affected.
  - **Description:** A detailed explanation of the change.

Example:
```
feat(authentication): add login validation
```

## Pull Request Procedures
1. Ensure your branch is up to date with the base branch.
2. Submit your pull request for review.
3. Be open to feedback and willing to make necessary changes.
4. Once approved, the pull request will be merged into the main codebase.

Thank you for contributing!
